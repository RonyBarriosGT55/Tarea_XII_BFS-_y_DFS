Este proyecto consiste en una comparación técnica entre dos formas de implementar un mismo grafo y sus recorridos BFS y DFS. 
Por un lado, se analiza una versión en C++  construida completamente a mano con punteros y gestión manual de memoria dinámica
para entender cómo funcionan los datos a bajo nivel. Por el otro, se presenta una versión funcional en Java que aprovecha
herramientas modernas como HashMap y Queue para agilizar el desarrollo y automatizar la limpieza de la memoria a través del 
Recolector de Basura. Al ejecutar el programa, el sistema procesa el grafo y despliega en consola los recorridos
correspondientes, demostrando de forma práctica el balance entre el máximo rendimiento de C++ y la seguridad y
comodidad que ofrece Java para el desarrollo de software real.
